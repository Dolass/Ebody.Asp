 版本: 1.1.1
*******************************************************************************
 简介:
   对于日渐繁多的数据，通常都需要对这些数据做一些基本的控制(新增，修改，删除)。如果写代
   码，那将是一个另人头疼而痛苦的事情。
   那么，有没有一个通用的控制器，只要连上数据库，通常要用到的操作功能就已经具备呢?
   EasyBodyShow(简称:Ebody)针对此需求应运而生。他基本组成是使用一个个类，也就是上面所称的
   控制器。有了他，你的开发过程将变得很轻松简单，另人头疼的数据控制编写过程将不复存在。
   我们的目标是傻瓜式的开发。

 特点:
   Ebody能为你节省编写数据控制程序的时间，摆脱繁琐的程序开发过程，将精力集中在逻辑分析和
   界面设计中去。
   面向对像的程序设计思想及遵循MVC架构开发方式，使系统的开发更简单高效，易用性强。
   性能优化的内核，能简单快速的读写数据和显示数据。
   友好灵活的二次开发语言，只要你会SQL语言，你就能轻松上手。
   支持所见即得的界面的设计方式，简单直观。自由定义显示界面，逻辑与界面完全分离，界面设计者
   再也不用去思考数据的来源问题，
   你想怎么显示数据，只需在你要显示的地方加上定义好对应数据库栏的标签，如何排版由你定。
   支持生成静态文件。

 不足:
   1. 数据输入栏位暂不支持中文(即SQL中的栏位别名不能为中文，否则取不到正确的值)
   2. 数据导出功能不稳定，不支持混合(简繁体字符)导出
   3. 数据分页功能不完善

 适用平台: MS IIS 5.0以上
 适用产品: 本程序适用于小型管理系统的开发，网站信息发布等
 适用人群: 有一定SQL基础的人员
 开发语言: ASP(VBscript)
 支持的DB: Excel，Access，SQLServer，Oracle(或更多)

 系统二次开发约定
*******************************************************************************

  程序代码命名---------------------------
  原则: 	范围(小写) + 类型(小写) + 功能名称(各单词首字大写)
        单词间直接连接,不用任何分隔符
  如: lvUserName 正确, lv_User_Name 错误
  范围如下
  c: 	Class,类中定义
  l: 	Local Variable,局部变量，如过程，函数内部使用
  类型如下
  o: 	Object,对像
  v: 	Variable,变量
  s:	State,常量

  文件命名---------------------------
  公司名(2位) + 模块名(3位) + 功能代号(1位) + 流水号(3位)
  公司名: 取公司的简称,如NetCom,则为NC

  模块名包括(示例)	
  INV 库存	
  WIP 生产	
  PUR 采购	
  FIN 财务	
  OEM 订单	
  SYS 系统
  ...		

  功能代号包括(示例)
  R Report报表查询
  B Base基本资料
  I Import数据导入
  E Export数据导出
  M Manager数据管理
  ...                  

  流水号: 每个公司,模块,功能下的流水号独立
  如: NCINVR001 江波龙库存管理查询报表
      NCPURB001 江波龙采购管理基本资料

  表命名---------------------------
  表及栏位命名所用的单词分别用_划分
  如: User_Name 正确, UserName 错误

  代码注释及消息---------------------------
  所有注释内容不管中英文都使用半角（英文式）标点符号
  在提示给用户的消息的内容存在中文的使用全角（中文式）标点符号
